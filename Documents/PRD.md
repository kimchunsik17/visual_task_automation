## TL;DR (요약)

사용자가 시각적인 캔버스에서 노드를 연결하여 비즈니스 자동화 워크플로우를 설계하고 실행할 수 있는 파일럿 툴입니다. 프론트엔드(React Flow)에서 그린 그래프 데이터를 백엔드(LangGraph + Gemini)가 해석해 AI 기반의 작업 요약 및 실행 결과를 반환합니다. 목표는 코딩 없이 직관적인 UI로 AI 에이전트 워크플로우를 테스트하고 검증하는 환경을 구축하는 것입니다.

## 1. 배경 & 문제

- **현상**: 비즈니스 자동화 및 AI 에이전트 연동이 점점 중요해지고 있으나, 이를 구성하려면 복잡한 코드 작성과 백엔드 파이프라인(LangChain/LangGraph 등) 이해가 필요합니다.
- **문제 정의**: 비개발자나 기획자도 손쉽게 AI 워크플로우를 구상하고 테스트해볼 수 있는 직관적인 시각화 도구(노드 기반 UI)가 부재합니다.

## 2. 목표 & 성공 지표

- **목표**: 
  1. 노드 드래그 앤 드롭으로 워크플로우를 구성하고 즉시 실행해 볼 수 있는 환경 제공.
  2. 프론트엔드(React Flow)와 백엔드(LangGraph) 간의 매끄러운 그래프 데이터 변환 및 통신 검증.
- **지표**:
  - **테스트 워크플로우 실행 성공률**: 구성된 그래프의 정상 실행 및 결과 반환 비율 (목표 95%)
  - **UI 반응성**: 노드 연결 및 렌더링 지연 시간 (목표 200ms 이하)

## 3. 비목표 (Non-Goals)

- 복잡한 다중 분기(Condition) 로직이나 반복문(Loop) 구현 (파일럿에서는 순차 연결 위주로 검증).
- 상용 수준의 엔터프라이즈 권한 관리 및 결제 연동.
- 자체 LLM 모델 구축 (외부 API인 Gemini에 의존).

## 4. 대상 사용자 & 사용자 스토리

| 페르소나 | 상황 | 핵심 니즈 |
| --- | --- | --- |
| 프로세스 기획자 | 자동화 아이디어를 빠르게 테스트하고 싶음 | 코딩 없이 노드를 이어 붙여 AI 워크플로우 실행 |
| 백엔드 개발자 | LangGraph와 프론트엔드의 연동을 테스트 중 | 프론트엔드에서 넘어온 JSON 그래프가 정상 작동하는지 확인 |

- **US-1**: 기획자로서, 캔버스에 'Start Trigger'와 'Generate Summary Task' 노드를 배치하고 연결하고 싶다.
- **US-2**: 기획자로서, 'Run Flow' 버튼을 눌러 연결된 AI 워크플로우의 실행 결과를 우측 패널에서 확인하고 싶다.

## 5. 핵심 사용자 흐름 (User Flow)

웹 접속 → 캔버스에 노드(Start, Task) 배치 → 엣지(Edge)로 노드 연결 → 'Run Flow' 실행 → 그래프 데이터(JSON) 백엔드 전송 → LangGraph+Gemini 처리 → 실행 결과 패널 표시

## 6. 기능 요구사항

| ID | 기능 | 우선순위 |
| --- | --- | --- |
| F-1 | 캔버스 UI 및 노드 드래그 앤 드롭 (React Flow) | P0 |
| F-2 | 노드 간 연결(Edge) 생성 및 삭제 | P0 |
| F-3 | 워크플로우 실행 (Run Flow) API 연동 | P0 |
| F-4 | 실행 결과(LangGraph+Gemini 응답) 화면 표시 | P0 |
| F-5 | 다중 AI 작업 노드 추가 지원 | P1 |

## 7. AI 에이전트 요구사항

- **자율 수준**: 워크플로우형(사용자가 연결한 노드 순서대로 실행).
- **역할**: LangGraph가 프론트엔드의 노드 흐름을 해석하여 Gemini API를 호출, 워크플로우의 의도나 요약 결과를 생성.
- **가드레일**: 과도한 API 호출 방지를 위해 노드 개수 및 순환 참조(Cycle) 제한.

## 8. 비기능 요구사항 (NFR)

- **성능**: 프론트엔드 캔버스 조작 시 끊김 없는 60fps 렌더링 (React Flow 최적화).
- **호환성**: Chrome, Safari 등 모던 데스크톱 브라우저 지원.
- **확장성**: 추후 다양한 종류의 노드(API 호출, DB 조회 등)를 쉽게 추가할 수 있는 백엔드 구조.

## 9. 의존성 & 가정

- **의존성**: Gemini API, React Flow 라이브러리, LangGraph, PostgreSQL.
- **가정**: 사용자는 노드 기반 도구(예: Zapier, ComfyUI 등)에 대한 기본적인 이해도가 있다.

## 10. 리스크 & 완화

- **Gemini API 지연/오류**: 외부 API 장애 시 "실패" 상태를 반환하고, 재시도 로직 구현.
- **무한 루프(Cycle) 오류**: 사용자가 노드를 순환 형태로 연결할 경우, 그래프 검증 로직을 통해 실행 전 차단.
