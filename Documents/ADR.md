## ADR-0001 · 워크플로우 엔진: LangGraph 채택

| 상태 | 수락됨 · 2026-07-10 |
| --- | --- |
| 결정자 | 김춘식, 백엔드 |

**맥락 (Context)**

사용자가 그린 노드-엣지 그래프를 백엔드에서 순차적 혹은 조건부로 실행할 수 있는 AI 워크플로우 엔진이 필요하다.

**결정 (Decision)**

우리는 에이전트 상태를 그래프로 관리하는 데 특화된 **LangGraph**를 채택한다.

**고려한 대안 (Considered Options)**

| 대안 | 장점 | 단점 |
| --- | --- | --- |
| LangGraph (채택) | 순환 구조 및 상태 전이 제어 탁월 | 상대적으로 최신 라이브러리로 학습 필요 |
| 자체 DAG 로직 구현 | 완벽한 커스터마이징 | 예외 처리 및 AI 상태 관리에 막대한 개발 리소스 소요 |

**결과 (Consequences)**

- 좋아지는 것: 복잡한 AI 에이전트 플로우를 상태(State) 기반으로 손쉽게 관리 가능.
- 감수: 프론트엔드 그래프 모델을 LangGraph 스키마로 번역하는 어댑터 로직 구현 필요.

## ADR-0002 · 프론트엔드 UI: React Flow 채택

| 상태 | 수락됨 · 2026-07-10 |
| --- | --- |
| 결정자 | 김춘식, 프론트엔드 |

**맥락 (Context)**

워크플로우 시각화 도구의 핵심은 노드 배치와 연결(Edge)을 쉽게 할 수 있는 캔버스 환경이다.

**결정 (Decision)**

우리는 React 기반의 노드 캔버스 라이브러리인 **React Flow**를 채택한다.

**고려한 대안 (Considered Options)**

| 대안 | 장점 | 단점 |
| --- | --- | --- |
| React Flow (채택) | 압도적인 커뮤니티, 리액트 컴포넌트로 노드 확장 용이 | 대규모 노드 시 최적화 필요 |
| D3.js 기반 직접 구현 | 세밀한 시각화 제어 가능 | 드래그, 줌, 연결선 등 기본 기능 구현에 시간 소요 |

**결과 (Consequences)**

- 좋아지는 것: 줌, 팬, 드래그 기능이 내장되어 있어 빠른 파일럿 UI 개발 가능.
- 감수: React Flow의 JSON 포맷에 강하게 결합됨.

## ADR-0003 · LLM 제공: Gemini API 채택

| 상태 | 수락됨 · 2026-07-10 |
| --- | --- |
| 결정자 | 김춘식 |

**맥락 (Context)**

LangGraph 내의 태스크 노드를 실행하고 요약 결과를 내기 위한 LLM이 필요하다.

**결정 (Decision)**

우리는 빠르고 효율적인 AI 추론을 위해 **Gemini API**를 사용한다.

**고려한 대안 (Considered Options)**

| 대안 | 장점 | 단점 |
| --- | --- | --- |
| Gemini API (채택) | 빠른 응답 속도, 강력한 추론 능력 | 프롬프트 미세 조정 필요 |
| 오픈소스 로컬 모델 | 비용 무료 | 인프라 셋업 부담 및 속도 저하 |

**결과 (Consequences)**

- 좋아지는 것: 인프라 고민 없이 비즈니스 로직(워크플로우 구성)에 집중할 수 있음.
- 감수: `.env`를 통한 키 관리 필요.

## ADR-0004 · 데이터베이스: PostgreSQL 채택

| 상태 | 수락됨 · 2026-07-10 |
| --- | --- |
| 결정자 | 김춘식, 백엔드 |

**맥락 (Context)**

향후 워크플로우 템플릿과 실행 이력을 저장하기 위해 관계형 DB이면서 JSON 포맷을 유연하게 저장할 수 있는 스토리지가 필요하다.

**결정 (Decision)**

우리는 JSONB 타입을 강력하게 지원하는 **PostgreSQL**을 채택한다.

**고려한 대안 (Considered Options)**

| 대안 | 장점 | 단점 |
| --- | --- | --- |
| PostgreSQL (채택) | 정형 데이터와 비정형 데이터(JSONB) 동시 처리 용이 | 셋업에 RDBMS 지식 필요 |
| MongoDB | JSON 문서 그대로 저장하기 적합 | 향후 트랜잭션 등 복잡도 증가 시 대응 불편 |

**결과 (Consequences)**

- 좋아지는 것: 프론트에서 온 Graph JSON을 그대로 저장하면서, 유저 정보는 정규화하여 관리 가능.
- 새 리스크: DB 스키마 마이그레이션 관리 필요(Alembic 등 도입 예정).
